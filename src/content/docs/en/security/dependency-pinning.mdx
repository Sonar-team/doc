# ЁЯЫбя╕П Step 1 тАФ Dependency pinning

## ЁЯОп Goal

Ensure that every compilation of SONAR relies on a **strictly determined,
reproducible and auditable set of dependencies**, regardless of:

* the build machine
* network connectivity
* the moment of compilation

Without this step:

* the build is **not deterministic**
* the chain of trust is **broken**
* the (Cosign) signature loses its value

---

## ЁЯУж Scope

Pinning covers:

* Rust dependencies (direct + transitive)
* Deno dependencies (remote modules)
* version resolution
* dependency sources
* build configuration

---

## ЁЯФР Mandatory rules

### 1. Mandatory Rust lockfile

The `Cargo.lock` file must:

* be **present**
* be **versioned**
* never be modified implicitly
* not contain '^' in version numbers

all versions are fixed (no ^, no ~)
тЖТ so no automatic supply-chain drift

```bash
cargo build --locked
```

Recommended strict mode:

```bash
cargo build --frozen
```

---

### 2. No dynamic resolution

Every compilation must be:

* **without network access**
* **without implicit updates**

ЁЯСЙ Any download attempt must fail the build

---

### 3. Controlled dependency updates

Updates must be:

* explicit (`cargo update`)
* traced (dedicated commit)
* validated (review)
* tested offline

---

## ЁЯжА Rust тАФ Implementation

### Generating the lockfile

```bash
cargo generate-lockfile
```

---

### Prefetching dependencies

```bash
cargo fetch --locked
```

---

### Vendoring (mandatory for offline)

```bash
cargo vendor vendor > .cargo/config.toml
```

---

### Expected configuration

```toml
[source.crates-io]
replace-with = "vendored-sources"

[source.vendored-sources]
directory = "vendor"
```

---

### Secure build

```bash
cargo build --frozen
```

---

## ЁЯЯб Deno тАФ Implementation

### Generating the lockfile

Create the `deno.json` file with the following elements:

```json
{
  "vendor": true,
  "lock": {
    "frozen": true
  },
  "nodeModulesDir": "auto"
}

```
so:
vendor: true тЖТ you cache dependencies locally тЖТ excellent for air-gapped
lock.frozen: true тЖТ you prevent any dependency drift тЖТ very good

---

### Prefetching the cache

```bash
set DENO_DIR=.deno-cache
deno cache deno.json
```
deno install --frozen
---

## тЪая╕П Deno constraints

The build must:

* contain **no unpinned dynamic import**
* never trigger an implicit download

---

## ЁЯУБ Artifacts to version

Mandatory in the repository:

* `Cargo.lock`
* `.cargo/config.toml`
* `vendor/`
* `deno.lock`
* `deno.json`

---

## тЭМ Prohibitions

* deleting `Cargo.lock`
* building without `--locked` or `--frozen`
* depending on non-vendored crates
* using Deno without `--cached-only`
* importing unpinned dynamic dependencies

---

## тЬЕ Validation criteria

The step is validated if:

* build succeeds **without network**
* no download attempt
* `cargo build --frozen` passes
* no modification of `Cargo.lock` during the build

## ЁЯза Architecture decision

Chosen option:

тЮбя╕П **Targeted vendoring (`cargo vendor`)**

Reasons:

* simpler to audit
* reproducible
* compatible with strict offline
* no need for a crates.io mirror

* dependency traceability
* resistance to supply-chain attacks
* a reliable basis for Cosign signing
